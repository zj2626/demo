索引:
    优点: 提高少量数据的获取/检索速度
    缺点: 1.占用存储空间(复合索引占用的更大)
          2.多个索引挑选耗费时间
          3.降低写操作效率,要实时维护索引
          4.并发下索引维护复杂度高

    索引的不使用场景:
        1.数据重复度高
        2.选择率高于10%
        3.表的数据量少

    索引对where: 当查询数据满足上面的不使用场景,就查询优化器就不会使用索引,即使where条件中有加索引
    索引对查询列名称: 查询结果列名尽量也加索引,这样就不需要读取数据文件,减少IO操作
    索引对表达式计算: 索引列不能用于表达式(where ID+1=2),否则就不使用索引而全表扫描
    索引对内连接/外连接:
        1.当索引列作为连接条件:使用索引定位被连接的元组(连接和被连接的元组最好都有索引)
        2.数据值对于外连接消除有影响
    索引对分组: 使用索引列进行分组可以使用到索引(如果分组的索引列上有函数操作则不行)
                如果没有索引则会利用文件排序(Using temporary; Using filesort)
                如果使用了组合索引的但是不包含其中第一个子部分,则会进行文件排序
    索引对排序: 使用索引列进行排序可以使用到索引(如果排序的索引列上有函数操作则不行)
                如果没有索引则会利用文件排序(Using temporary; Using filesort)
                使用主键索引/组合索引才会使用到索引
                如果使用了组合索引的但是不包含其中第一个子部分,则会进行文件排序
    索引对去重: 使用索引列进行distinct可以使用到索引(如果去重的索引列上有函数操作则不行)
    索引对函数: 使用索引列进行函数操作可以使用到索引
                如果使用了组合索引的但是不包含其中第一个子部分,也可以使用上索引
