子查询:    一个查询是另一个查询的子部分
    分类:
        1. 从对象角度
            1.相关子查询: 子查询执行依赖于外部查询的一些属性值,多数情况下是子查询的WHERE子句中引用了外部查询的表
            2.非相关子查询: 不依赖外部; 执行先于父查询
        2. 从特定谓词
            1.[NOT] IN/ALL/ANY/SOME 子查询 : 其左边是操作数,右边是子查询
            2.[NOT] EXISTS 子查询(半连接语义): 没有左操作数, 右边是子查询
            3.其他子查询
        3. 从语句复杂程度
            1.SPJ子查询    :子查询语句由选择,连接,投影组成的子查询
            2.非SPJ子查询  :
                1.GROUPBY子查询:SPJ子查询加上分组,聚集操作
                2.其他子查询
        4. 从结果角度
            1.标量子查询: 子查询部分只能得到单条结果,是一个简单值
            2.单行单列子查询: 子查询返回结果集是零条或一条元组(0个或者1个简单值)
            3.多行单列子查询: 返回的结果集是多条元组但是只有一个简单列
            4.表子查询: 子查询返回一个表(多行多列)


查询优化技术类型:
    1. 语法级:查询语法层的优化,基于语法进行
    2. 代数级:运用代数关系原理进行优化
    3. 语义级:(Semantic.txt),根据完整性约束,对查询进行语义理解
    4. 物理级:基于代价估算模型,变化不同的执行方式

    查询重写是基于1,2,3的优化,属于逻辑优化;物理层面优化是从连接路径中选择代价最小路径的过程

查询优化技术依据 三个

    2. 关系代数         :数学规则理论依据,公式推导
    3. 启发式规则        :实践经验
